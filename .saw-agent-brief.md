## Task

Implement upgrade logic for converting Agent Skills to progressive
disclosure pattern.

**What to implement:**

Create `src/upgrade/mod.rs` as the upgrade module root:
- Re-export submodules (analyzer, splitter, generator)
- Define `pub fn upgrade_skill(skill_path: &Path, options: &UpgradeOptions) -> Result<(), Error>`
- Call analyzer to get BloatAnalysis
- If dry_run: print analysis + suggested changes to stdout, return Ok(()) early
- Call splitter to get SplitResult
- Call generator to create inject-context script content
- Write files: update SKILL.md, create references/ dir + files, create scripts/inject-context
- Set inject-context executable using `std::fs::set_permissions` with mode 0o755 (Unix only, no-op on Windows)

Create `src/upgrade/analyzer.rs`:
- `pub struct BloatAnalysis` (total_lines, suggested_splits, trigger_patterns, needs_agent_references)
- `pub struct SplitSuggestion` (section_name, start_line, end_line, target_file)
- `pub fn analyze_bloat(skill_path: &Path) -> Result<BloatAnalysis, Error>`
- Read SKILL.md, count lines
- Detect markdown sections using regex (## headers)
- Heuristics for split suggestions:
  * Sections with "Reference", "Procedure", "Implementation", "Steps" in title → move to references/
  * Sections >50 lines → move to references/
  * Code blocks (triple backticks) >30 lines → move to references/
- Generate trigger patterns from skill name (extract from frontmatter) + common prefixes (e.g., `/skillname`, `skillname:`)
- Set `needs_agent_references: true` if UpgradeOptions.with_agent_references flag set

Create `src/upgrade/splitter.rs`:
- `pub struct SplitResult` (core_content, reference_files, triggers)
- `pub fn split_content(skill_path: &Path, analysis: &BloatAnalysis) -> Result<SplitResult, Error>`
- Read SKILL.md content
- For each SplitSuggestion: extract section lines, remove from core, add to reference_files map
- Add dedup markers to start of each reference file content: `<!-- injected: references/filename.md -->\n`
- Generate triggers frontmatter YAML from analysis.trigger_patterns
- Prepend triggers to core_content as YAML frontmatter block (between `---` delimiters)
- If needs_agent_references: add `agent-references:` field to frontmatter listing reference files

Create `src/upgrade/generator.rs`:
- `pub fn generate_inject_script(skill_path: &Path, reference_files: &[String]) -> Result<String, Error>`
- Generate bash script content with:
  * Shebang `#!/usr/bin/env bash`
  * Comments explaining purpose and auto-generation
  * Logic to detect SKILL_DIR (directory containing SKILL.md)
  * For each reference file: inject content into agent context before SKILL.md is delivered
  * Dedup check: if agent context already contains `<!-- injected: references/X.md -->`, skip injection
- Script template structure:
  ```bash
  #!/usr/bin/env bash
  # Auto-generated by agentskills upgrade
  # Injects references/ content into agent context before SKILL.md delivery

  SKILL_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

  for ref in "$SKILL_DIR"/references/*.md; do
    marker="<!-- injected: $(basename "$ref") -->"
    if ! grep -q "$marker" "$AGENT_CONTEXT" 2>/dev/null; then
      cat "$ref" >> "$AGENT_CONTEXT"
    fi
  done
  ```

## Interface Contracts

### SkillMetadata

Parsed representation of Agent Skill SKILL.md frontmatter. Contains
required fields (name, description) and optional extension fields
(triggers, agent-references, model, model-context, version).


```
pub struct SkillMetadata {
    pub name: String,
    pub description: String,
    pub triggers: Option<Vec<String>>,
    pub agent_references: Option<Vec<String>>,
    pub model: Option<String>,
    pub model_context: Option<String>,
    pub version: Option<String>,
    pub unknown_fields: HashMap<String, serde_yaml::Value>,
}

impl SkillMetadata {
    pub fn from_path(path: &Path) -> Result<Self, Error>;
    pub fn from_str(content: &str) -> Result<Self, Error>;
}

```

### ValidationError

Represents a single validation error or warning. Contains error type,
message, optional file location, and severity level (error vs warning).


```
#[derive(Debug, Clone, PartialEq)]
pub struct ValidationError {
    pub error_type: String,
    pub message: String,
    pub file: Option<PathBuf>,
    pub line: Option<usize>,
    pub severity: Severity,
}

#[derive(Debug, Clone, PartialEq)]
pub enum Severity {
    Error,
    Warning,
}

```

### ValidationResult

Aggregates validation errors and warnings. Provides methods to check
if validation passed (no errors) and to format output for CLI display.


```
#[derive(Debug, Default)]
pub struct ValidationResult {
    pub errors: Vec<ValidationError>,
    pub warnings: Vec<ValidationError>,
}

impl ValidationResult {
    pub fn new() -> Self;
    pub fn add_error(&mut self, error: ValidationError);
    pub fn add_warning(&mut self, warning: ValidationError);
    pub fn is_valid(&self) -> bool;
    pub fn format_output(&self) -> String;
}

```

### validate_skill

Main validation entry point. Runs base spec checks, extension validation,
and progressive disclosure checks. Returns aggregated ValidationResult.


```
pub fn validate_skill(path: &Path) -> Result<ValidationResult, Error> {
    let metadata = SkillMetadata::from_path(path)?;
    let mut result = ValidationResult::new();

    validate_base_spec(&metadata, &mut result);
    validate_extensions(&metadata, &mut result);
    validate_progressive_disclosure(path, &mut result)?;

    Ok(result)
}

```

### validate_base_spec

Checks that required fields (name, description) are present and non-empty.
Verifies SKILL.md file exists.


```
pub fn validate_base_spec(metadata: &SkillMetadata, result: &mut ValidationResult);

```

### validate_extensions

Validates known extension fields (triggers format, agent-references format,
model values). Warns on unknown fields without blocking (encourages innovation).


```
pub fn validate_extensions(metadata: &SkillMetadata, result: &mut ValidationResult);

```

### validate_progressive_disclosure

Checks progressive disclosure patterns: SKILL.md length (<200 lines),
references/ structure, inject-context script existence, dedup markers.


```
pub fn validate_progressive_disclosure(skill_path: &Path, result: &mut ValidationResult) -> Result<(), Error>;

```

```

pub fn analyze_bloat(skill_path: &Path) -> Result<BloatAnalysis, Error>;
```
